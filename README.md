# Tony Skyview — 드론 영상 소개 페이지

## 프로젝트 소개

개인 드론으로 찍은 제주 주변 사진을 모아 두고, **넷플릭스 메인 화면처럼 어둡고 카드형**으로 보여 주는 연습용 웹 페이지입니다.  
실제 영상 재생 기능은 없고, **HTML과 CSS만**으로 레이아웃과 분위기를 맞춰 본 프로젝트입니다.

## 파일 구성

| 파일·폴더 | 역할 |
|-----------|------|
| `index.html` | 페이지 구조(헤더, 배너, 카드, 푸터) |
| `styles.css` | 전체 스타일·반응형(미디어 쿼리) |
| `images/` | 배너·카드용 사진(`songak.png`, `sea_cave.jpg`, `daepo.jpg` 등) |

## 사용 기술

- HTML5 (`header`, `main`, `section`, `footer`, `article` 등 시맨틱 태그)
- CSS3  
  - 플렉스 레이아웃, 호버, 하단 그라데이션(`::after`)  
  - **미디어 쿼리**(768px 이하에서 타이포·패딩·카드 열 수 조정)  
  - 배너: 데스크톱은 비율 유지, 모바일은 `object-fit: cover`와 고정 높이  
  - 글자 가독성: **겹친 `text-shadow`**, 제목·부제 색 살짝 구분  
- JavaScript는 사용하지 않습니다.

## 구현한 화면 구성

| 영역 | 내용 |
|------|------|
| **헤더** | 로고 **TONY SKYVIEW**, 부제「개인 드론 영상 모음」 |
| **배너(히어로)** | `songak.png` + 송악산 제목·설명(하단 그라데이션·그림자로 읽기 쉽게) |
| **추천 영상** | 섹션 제목「추천 드론 영상」(카드 제목보다 크고 굵게) |
| **카드 ×5** | 해식동굴, 대포주상절리대, 성산일출봉, 범섬 해식동굴, 갯깍주상절리대 |
| **푸터** | `© 2026 Tony Skyview. HTML/CSS Practice Project.` |

### 데스크톱과 모바일에서 배너가 다른 점

| 구분 | 배너 이미지 |
|------|-------------|
| **769px 이상(데스크톱)** | `width: 100%`, `height: auto`로 **파일 비율 그대로**, 넓은 히어로 느낌 |
| **768px 이하** | 카드 한 장(16:9 썸네일 + 제목·설명) 높이에 맞춘 **고정 높이** + `object-fit: cover`로 **잘라서** 채움 |

모바일·태블릿(세로)에서는 카드와 높이가 비슷해 보이도록 맞추고, 데스크톱에서는 예전처럼 **풍경이 넓게** 보이도록 나눠 두었습니다.

### 모바일(768px 이하) 공통

- 로고·배너·섹션 글자 크기와 패딩 조정, 히어로 제목에 `clamp`·`line-height`
- 카드 **한 줄에 하나**(너비 100%에 가깝게)
- **400px 이하**에서 히어로 제목만 한 번 더 작게

이미지 경로는 실제 파일명에 맞춤(예: 동굴 썸네일은 `images/sea_cave.jpg`).

## 작업하면서 수정한 내용 (시간 순 느낌)

1. **첫 레이아웃**  
   배너에 `background-image` + `cover`를 쓰다가 송악산이 **위·아래 잘리는** 문제가 있어, `contain`·높이 제한도 시험해 봤습니다.

2. **배너를 `<img>`로**  
   박스 높이와 `overflow` 때문에 계속 애매해서, **`songak.png`를 `<img class="banner-img">`로 두고** 데스크톱에서는 `height: auto`로 **전체 풍경**이 보이게 정리했습니다.

3. **카드 5개·문구 표 반영**  
   `tafoni.jpg` 추가, 장소별 제목·설명 수정. 로고 **TONY SKYVIEW**, 푸터 **영문 한 줄**로 통일.

4. **모바일 대응**  
   제목 겹침·답답함을 줄이기 위해 **768px 이하** 미디어 쿼리로 글자·여백·카드 열 조정. **HTML은 거의 그대로, CSS 중심.**

5. **가독성·위계**  
   - 히어로: 반투명 **배경 박스**는 쓰지 않고, **테두리 느낌 `text-shadow`**와 부제 색(`#f5f0e8`)만으로 대비.  
   - 「추천 드론 영상」은 **더 크고 굵게**(`row-title`), 카드 제목(`h3`)과 구분.

6. **배너 높이 실험**  
   모바일에서 카드와 높이를 맞추려 **고정 높이 + `cover`**를 썼다가, 같은 규칙이 데스크톱까지 적용되어 배너가 **너무 납작**해 보였습니다.  
   그래서 **769px 이상은 예전처럼 비율 유지**, **768px 이하만** 카드 높이에 맞춘 방식으로 **다시 나눴습니다.**

7. **겹침 순서**  
   배너에서 이미지·그라데이션·글자가 겹칠 때를 위해 **`z-index`**로 순서를 정해 두었습니다(모바일에서 이미지가 `absolute`일 때 포함).

## AI를 활용한 방식

과제 조건과 화면 느낌을 **짧은 프롬프트**로 여러 번 나눠 요청했습니다.  
브라우저·폰 화면을 보며 잘림, 겹침, 글자 대비 같은 **눈에 보이는 문제**를 구체적으로 적어 수정을 이어 갔습니다.  
구조를 어느 정도 고정한 뒤에는 **텍스트만** 고치거나 **CSS만** 손대는 식으로 범위를 줄였습니다.  
모바일 문제는 **스크린샷**을 기준으로 미디어 쿼리 조건을 정했습니다.

## 실행 방법

1. 이 폴더를 그대로 둡니다.
2. **`index.html`** 을 더블클릭하거나 브라우저에서 **파일 열기**로 연다.
3. `styles.css`와 `images` 폴더가 `index.html`과 같은 위치에 있어야 합니다.

별도 서버나 빌드는 없습니다.  
**너비별 확인**: 개발자 도구(F12)에서 기기/반응형 모드로 너비를 바꿔 보거나, 실제 폰에서 같은 파일을 열어 보면 됩니다.

## 느낀 점

- “꽉 차게”와 “사진 전체 보이게”는 **서로 다른 목표**라서, 같은 코드로 둘 다 만족하기 어렵다는 걸 배너에서 많이 느꼈습니다.  
- **화면 너비별로 배너 규칙을 나누는 것**이 데스크톱·모바일 모두 만족스럽게 가는 지름길이었습니다.  
- 반투명 박스 없이도 **`text-shadow`와 글자색**만으로 읽기 좋게 만들 수 있다는 것도 알게 되었습니다.  
- 시맨틱 태그로 영역을 나눠 두니 수정할 때 찾기 편했고, 데스크톱만 보다가 모바일 **줄 간격** 문제를 놓치기 쉽다는 것도 체감했습니다.
